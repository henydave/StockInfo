Rasa Pizza bot: Mini Project from Scratch within 30 minutes

Heny Dave
Jun 5 · 6 min read

I wanted to build a chatbot smart enough to deal with single to multiple queries and can even handle typos and abbreviations with ease. So, here I made a chatbot which uses real Machine Learning and NLP techniques to make communications feel like as if talking to a real human using an open source framework Rasa. Before jumping into implementation lets first get started with setup on your machine. Make sure to use Ubuntu version 18.04 for it. Using different version of Ubuntu or Windows or MacOS will give dependency or installation errors.

  1. Virtual Environment
    You can make virtual environment by installing Anaconda. After installing Anaconda, run the below steps from terminal.
conda create --name rasa python=3.6
conda activate rasa  //Use this command always after opening a new terminal.

2. Now clone/download my GitHub repository into your folder and install all the requirements.

(rasa)$ cd path_to_the_project_folder
(rasa)$ pip install -r requirements.txt

The installation of requirements will take some time. So, till then lets go through the basics of Rasa, and how actually it works.

If you are familiar with basic terminologies of Rasa, skip the below topics and jump here.

3. Now lets start with the implementation and learning

Rasa provides an easy and systematic approach to build a complete chatbot at your local desktop and its completely free. The main 2 Rasa terms are:

  • Rasa NLU: Rasa NLU deals with the user input by extracting entites from the user statement.

If user writes: I want a large pizza, then Rasa NLU extracts large as a size entity.

  • Rasa Core: A dialog management solution tries to build a probability model which decides the set of actions to perform based on the previous set of user inputs.

If user writes: I want a cheese pizza, then the next reply by the bot should be: What size pizza would you like ? and vice versa.

Some keywords you will find repeatably used in the post in reference to Rasa functions/tools,

  • Intent: Any sentence/word asked or stated by the user is an intent. If a user say, “I want to order a pizza”, the intent would be ordering pizza after asking about its specifications.
  • Entity: Consider it as the useful information from the user input that can be extracted. From previous example, by intent we understand the aim is to order a pizza, but of what size or toppings? If we extract “Pizza” as the entity, we can perform the action on pizza.
  • Actions: As the name suggest, its an operation which can be performed by the bot. It could be replying something in return, querying a database or any other thing possible by code.
  • Stories: These are a sample interaction between the user and bot, defined in terms of intents captured and actions performed. So developer can mention what to do if you get a use input of some intent with/without some entities. Like saying if user intent is to find the day of week and entity is today, find day of week of today and reply.

To make this work, Rasa need some files, which stores all the training and model information to build the bot. To give a brief overview, most important ones are,

  • NLU data file: It contains a bunch of examples of the user input along with their mapping to a suitable intent and entities present in each of them. The more varying examples you provide, better your bot’s NLU capabilities become. This file as the name suggests is a part of Rasa NLU. A sample file is embedded here:

Stories file: It contains many different sequences of possible dialogue conversations between the user and the bot. This file is a part of Rasa Core as it handles dialogues. A sample file is embedded here:
Here, utter_greet, utter_size, etc are the statements responded by the bot and provide_none, provide_size, etc represents text said or asked by the user.

Rasa has a feature of interactive learning which allows you to train bot and generate multiple stories which can easily be appended into stories.md file. More stories more smarter the bot.

  • Pipeline file: This file is heart of Rasa. Pipeline defines all various nlp processes the execution will pass through. Rasa also allows to use custom pipeline.
  • Domain file: Here you list all of the intents (user’s sentence), entities (important words extracted from intents like size, toppings of pizza ), actions (next action taken by bot), slots (contains no. of entities along with their value), templates (fixed statements asked by the bot), and similar information. You can also add sample bot reply templates and use them as actions.

After understanding the basis, lets move on to the implementation

I hope you would have created a virtual environment, cloned my github repository and installed all the requirements. So, lets move on.

1. First we need to train our Rasa NLU

make train-nlu

Training Rasa nlu takes nlu_data.md and the pipeline file as input and the generated trained data is saved into models/nlu folder.

2. Then train Rasa core

make train-core

Training Rasa core takes stories.md and domain.yml files as input and the data generated after training is saved into models/dialogue folder.

3. Now its time to run the bot

make run

After executing make run command, you have 2 ways to talk the bot.

  1. Talking through the terminal itself:
    You can see all the intents, next actions, current actions, slots, entities, etc selected by the bot to process the current input by the user.

2. Talking through the browser with a UI:
After running make run command in terminal (it should continue to run, don’t stop its execution), double click on ui/index.html file, which will open browser and you will see a small icon in bottom-right corner, click it and begin chatting with your newly created rasa bot ! :)

Make sure to execute make run command through a new terminal window always and closing the previously executed terminal windows, otherwise you will get an error as the localhost will be already running.

Interactive Learning
